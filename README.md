# Aplikasi Jadwal Kegiatan Harian Menggunakan Struktur Data B-Tree

Aplikasi Jadwal Kegiatan Harian ini merupakan sebuah program berbasis console (CLI) yang dirancang untuk membantu pengguna dalam mengelola aktivitas atau jadwal sehari-hari secara terstruktur dan terurut berdasarkan waktu. Program ini dibuat sebagai implementasi dari materi Struktur Data, khususnya penerapan struktur data B-Tree, yang ditulis secara manual tanpa menggunakan library atau modul eksternal. Seluruh kegiatan disimpan dalam sebuah file berformat JSON dan dimuat ke dalam struktur B-Tree setiap kali aplikasi dijalankan.

Aplikasi ini memiliki tujuan utama untuk memperlihatkan bagaimana struktur data B-Tree dapat diterapkan dalam skenario nyata. B-Tree dipilih karena memiliki kemampuan menyimpan data dalam kondisi terurut serta menyediakan waktu akses, pencarian, dan penyisipan yang sangat efisien meskipun jumlah data semakin besar. Dengan menggunakan key berupa kombinasi tanggal dan waktu, yaitu “YYYY-MM-DD HH:MM”, seluruh kegiatan yang dimasukkan ke dalam program akan otomatis tersimpan dalam urutan yang benar tanpa perlu dilakukan sorting tambahan. Hal ini menjadikan B-Tree sebagai pilihan yang sesuai untuk aplikasi penjadwalan yang membutuhkan keteraturan data.

Program ini terdiri dari beberapa komponen utama, yaitu model kegiatan, struktur node untuk tree, kelas B-Tree itu sendiri, kelas untuk validasi input, dan serializer yang bertugas membaca serta menyimpan data ke file JSON. Struktur folder program dibagi menjadi beberapa direktori seperti models, btree, utils, dan data, sehingga program lebih terorganisir dan mudah dipahami. Seluruh logika terkait struktur data dan manipulasi kegiatan dipusatkan pada file-file tersebut.

Saat aplikasi dijalankan, program pertama kali membaca semua data dari file jadwal.json dan memasukkannya ke dalam B-Tree. Setelah itu, pengguna diberikan menu utama untuk melakukan berbagai operasi seperti menambah kegiatan baru, mencari kegiatan, memperbarui kegiatan, menghapus kegiatan, atau menampilkan seluruh jadwal yang tersimpan. Proses penambahan kegiatan dilakukan dengan meminta input lengkap dari pengguna yang kemudian akan divalidasi. Jika format data sudah benar, maka data tersebut akan dimasukkan ke dalam B-Tree dan disimpan kembali ke dalam file JSON agar data tetap tersimpan meskipun program ditutup.

Pencarian kegiatan dapat dilakukan dengan dua cara, yaitu berdasarkan tanggal dan waktu tertentu untuk mencari satu kegiatan spesifik, atau berdasarkan tanggal saja untuk menampilkan semua kegiatan dalam satu hari. Program memanfaatkan traversal B-Tree untuk menampilkan kegiatan secara terurut, sehingga seluruh kegiatan pada hari yang sama ditampilkan berdasarkan urutan waktunya.

Fitur pembaruan kegiatan dilakukan dengan mencari data lama terlebih dahulu, menampilkannya kepada pengguna, kemudian meminta input data baru untuk menggantikannya. Karena fungsi delete pada B-Tree belum diimplementasikan, pembaruan dilakukan dengan cara menambahkan data baru, sementara data lama tetap tersimpan dalam file JSON. Penghapusan secara otomatis melalui B-Tree belum tersedia, sehingga pengguna yang ingin menghapus kegiatan harus melakukannya secara manual melalui file JSON.

Format penyimpanan data dalam JSON sangat sederhana. Setiap kegiatan terdiri dari ID, tanggal, waktu, nama kegiatan, tempat, dan deskripsi. Struktur ini memudahkan aplikasi dalam membaca dan menampilkan informasi kegiatan secara jelas dan terurut sesuai dengan key yang digunakan.

Aplikasi ini menunjukkan bagaimana struktur data tingkat lanjut seperti B-Tree dapat digunakan dalam aplikasi dunia nyata untuk menyimpan dan mengakses data secara efisien. Dengan sifatnya yang seimbang dan kemampuannya mempertahankan data dalam keadaan terurut, B-Tree menjadi fondasi yang kuat untuk sistem penjadwalan ini. Selain itu, penggunaan JSON sebagai media penyimpanan menjadikan aplikasi fleksibel dan mudah dikembangkan lebih lanjut.

Secara keseluruhan, aplikasi Jadwal Kegiatan Harian ini mampu melaksanakan fungsi-fungsi dasar penjadwalan dengan baik sekaligus memperlihatkan implementasi struktur data B-Tree secara praktis. Ke depannya, aplikasi ini masih dapat dikembangkan lebih jauh, seperti menambahkan fitur delete di dalam B-Tree, membuat antarmuka grafis, menambahkan kategori kegiatan, atau mengintegrasikannya dengan notifikasi pengingat. Program ini menjadi contoh penerapan nyata bagaimana konsep teori struktur data dapat digunakan untuk menyelesaikan masalah sehari-hari secara efektif.
